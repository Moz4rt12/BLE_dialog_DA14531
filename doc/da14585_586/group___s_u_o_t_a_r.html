◆ SUOTAR_SECURE_AUTO_REBOOT_TIMEOUT_MS

#define SUOTAR_SECURE_AUTO_REBOOT_TIMEOUT_MS   (0)

When set to a non-zero value, the device will not wait for a reboot command, and will reboot after SUOTA once the reboot timer expires or a disconnection occurs The auto-reboot timer will use this value as its timeout value (in ms)

◆ SUOTAR_SECURE_LOCK_FEATURE

#define SUOTAR_SECURE_LOCK_FEATURE   (1)

Enable/Disable the secure lock feature. When enabled, only ONE SUOTA session is allowed per boot Device will need to reboot after a SUOTA session in order to be able to perform another one
