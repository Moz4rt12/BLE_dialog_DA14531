__STATIC_FORCEINLINE void app_advertise_directed_start_op ( enum gapm_own_addr  address_src_type,
uint16_t  intv_min,
uint16_t  intv_max,
uint8_t  channel_map,
enum address_type  target_address_type,
uint8_t *  target_address,
uint8_t  ldc_enable 
)

Create advertising message for connectable directed event (ADV_DIRECT_IND). It supports the low duty cycle directed advertising mode.

Parameters
[in]address_src_typeThe source address type used during the advertising operation:
  • GAPM_STATIC_ADDR
  • GAPM_GEN_RSLV_ADDR
  • GAPM_GEN_NON_RSLV_ADDR
[in]intv_minMinimum interval for advertising.
[in]intv_maxMaximum interval for advertising.
[in]channel_mapThe channels used during the advertising operation.
[in]target_address_typeAddress type of the target device.
[in]target_addressAddress of the target device.
[in]ldc_enableEnable/disable low duty cycle mode.
  • 0 = disabled
  • 1 = enabled

Create advertising message for nonconnectable undirected event (ADV_NONCONN_IND).

Parameters
[in]address_src_typeThe source address type used during the advertising operation: GAPM_STATIC_ADDR, GAPM_GEN_RSLV_ADDR, GAPM_GEN_NON_RSLV_ADDR
[in]intervalThe advertising interval.
[in]channel_mapThe channels used during the advertising operation.
[in]advertise_modeThe advertising mode: GAP_NON_DISCOVERABLE, GAP_GEN_DISCOVERABLE, GAP_LIM_DISCOVERABLE, GAP_BROADCASTER_MODE
[in]adv_filt_policyThe advertising filter policy: ADV_ALLOW_SCAN_ANY_CON_ANY, ADV_ALLOW_SCAN_WLST_CON_ANY, ADV_ALLOW_SCAN_ANY_CON_WLST, ADV_ALLOW_SCAN_WLST_CON_WLST
[in]advertise_dataPointer to an array with the advertising data.
[in]advertise_data_lenThe length of the advertising data.
[in]scan_response_dataPointer to an array with the scan response data.
[in]scan_response_data_lenThe length of the scan response data.
Note
This function supports also the advertising with scan response (ADV_SCAN_IND), if the scan response data are NOT empty.
