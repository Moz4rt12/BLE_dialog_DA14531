ke_msg_id_t custom_msg_handler::id

Id of the handled message. Can also be KE_MSG_DEFAULT_HANDLER to direct all messages to one handler for the particular task_id The search is top down so KE_MSG_DEFAULT_HANDLER should be after other handlers for specific messages for the particular task

◆ task_id

ke_task_id_t custom_msg_handler::task_id

Id of the destination task. This is used because some messages have duplicate handlers like LLD_DATA_IND and LLD_STOP_IND. Only the type is taken into account and not the index for tasks that have multiple instances
