◆ adc_correct_sample()

uint16_t adc_correct_sample ( const uint16_t  adc_val)

Returns the corrected conversion result.

The raw conversion result is corrected:

  • Gain Error & Offset compensation, if Gain Error and Offset parameters are available,
  • Oversampling is taken into account.
    Parameters
    [in]adc_valRaw ADC value, read from the selected input channel, calling adc_get_sample. In the case of interrupt-driven measurements, the raw ADC value shall be obtained by reading the GP_ADC_RESULT_REG register directly, i.e. GetWord16(GP_ADC_RESULT_REG).
    Returns
    conversion result
    Note
    In the case of single-ended mode, actual input voltage = Vref * (conversion result) / (2^N - 1). In the case of differential mode, actual input voltage = (2 * Vref * (conversion result) / (2^N - 1)) - Vref. In both cases:
  • Vref may be 0.9V/1.8V/2.7V/3.6V,
  • N may be 10-16 bits, depending on the oversampling mode.
    See also
    adc_correction_apply
    adc_set_oversampling

◆ adc_get_temp_async()

void adc_get_temp_async ( uint16_t *  adc_samples,
uint16_t  len,
int8_t *  temp_values 
)

Returns temperature values based on raw ADC samples obtained by monitoring the ADC_INPUT_SE_TEMP_SENS input channel and then reading the GP_ADC_RESULT_REG register directly, i.e. GetWord16(GP_ADC_RESULT_REG).

Note
Shall be called in the case of interrupt-driven measurements:
  • single ADC interrupt,
  • continuous mode; DMA functionality disabled,
  • continuous mode; DMA functionality enabled.

Clears the ADC interrupt and calculates temperature values in degrees Celsius, either calibrated or uncalibrated (the latter when OTP calibration value is not present).

Parameters
[in]adc_samplesPointer to raw ADC samples.
[in]lenNumber of raw ADC samples.
[out]temp_valuesPointer to temperature values in degrees Celsius.
Warning
adc_start shall be called first.
See also
adc_start

Initializes and enables ADC.

Note
In the case of DA14531, adc_init applies the mandatory ADC settings, when the ADC_INPUT_SE_TEMP_SENS input channel is to be used.
In the case of DA14585/DA14586, the PD_PER domain shall be functional prior to ADC initialization: SetBits16(PMU_CTRL_REG, PERIPH_SLEEP, 0); while (!(GetWord16(SYS_STAT_REG) & PER_IS_UP));
Parameters
[in]cfgThe ADC configuration.
Note
Initialization does not take place if cfg is NULL.

◆ adc_set_oversampling()

__STATIC_FORCEINLINE void adc_set_oversampling ( uint8_t  oversampling_mode)

Sets oversampling mode.

When in oversampling mode, multiple successive conversions are executed and the results are averaged to increase the effective number of bits. The number of successively executed conversions is 2^(oversampling_mode).

Parameters
[in]oversampling_modeOversampling mode option (0-7).
